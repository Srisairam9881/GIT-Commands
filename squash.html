Git Squash Command

What is Git Squash?

Git squash is a technique used to combine multiple commits into a single commit. It is particularly useful for cleaning up commit history and creating concise, meaningful commits.

Git Squash Command

To squash multiple commits into a single commit, you can use an interactive rebase:

git rebase -i HEAD~<n>

This command opens an interactive rebase window where you can choose which commits to squash. Replace <n> with the number of commits you want to squash.

In the interactive rebase window, change the word "pick" to "squash" (or "s") for the commits you want to squash:

pick 1a2b3c Commit message 1
squash 4d5e6f Commit message 2
squash 7g8h9i Commit message 3

Save and close the rebase window. Git will then combine the selected commits into a single commit.

Example Usage

Let's say you have made several small commits while working on a feature branch, but you want to combine them into one meaningful commit before merging the branch into the main branch:

  1. Run an interactive rebase for the last 3 commits:
    2. git rebase -i HEAD~3
  2. In the interactive rebase window, change the commits to squash:
    3. pick 1a2b3c Implement feature A
squash 4d5e6f Fix bug in feature A
squash 7g8h9i Update documentation for feature A
  3. Save and close the rebase window.
  4. Git will combine the selected commits into one commit. You can edit the commit message if needed.
