Git Restore Command

What is Git Restore?

Git restore is a command in Git that allows you to restore files or directories in your working directory to their state at a specific commit or to discard changes made since the last commit. It is useful for undoing changes and reverting to a known state.

Git Restore Commands

To restore a specific file or directory to its state at a specific commit, you can use:

git restore --source=<commit> --staged --worktree <file>

This command restores the specified file or directory to its state at the given commit. The --staged option restores the file in the staging area, and the --worktree option restores it in the working directory.

To discard changes made to a specific file in the working directory and restore it to the state of the last commit, you can use:

git restore --source=HEAD --staged --worktree <file>

This command discards changes made to the specified file and restores it to the state of the last commit.

If you want to discard all changes made to the entire working directory and restore it to the state of the last commit, you can use:

git restore .

This command discards all changes made to the working directory and restores it to the state of the last commit.

Example Usage

Let's consider a scenario where you made some changes to a file but later realized that those changes were incorrect. To discard those changes and restore the file to its state at the last commit, you can use:

git restore --source=HEAD --staged --worktree <file>

This command will revert the file to its state at the last commit, effectively undoing the changes you made.
