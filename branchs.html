Git Branches and Commands

What are Git Branches?

In Git, a branch is a lightweight movable pointer to a commit. It allows you to work on different versions of your project concurrently, without affecting the main codebase. Branches are commonly used for feature development, bug fixes, and experimenting with new ideas.

Git Branch Commands

To list all branches in your repository, you can use:

git branch

This command will display a list of all branches in your repository, with an asterisk (*) indicating the current branch.

To create a new branch, you can use:

git branch <branch-name>

This command creates a new branch with the specified name.

To switch to a different branch, you can use:

git checkout <branch-name>

This command switches your working directory to the specified branch.

To create a new branch and switch to it in one step, you can use:

git checkout -b <new-branch-name>

This command creates a new branch with the specified name and switches to it.

To update the name of a branch, you can use:

git branch -m <old-branch-name> <new-branch-name>

This command renames the specified branch from old-branch-name to new-branch-name.

To delete a branch, you can use:

git branch -d <branch-name>

This command deletes the specified branch. Note that you cannot delete the branch you are currently on.

Merging Branches

To merge changes from one branch into another, you can use:

git merge <source-branch>

This command incorporates changes from the source-branch into the current branch.

Example Usage

Let's say you want to update the name of an existing branch from feature-branch to new-feature-branch:

git branch -m feature-branch new-feature-branch

Now, you want to merge changes from new-feature-branch into the main branch:

git checkout main
git merge new-feature-branch

This will merge the changes from new-feature-branch into the main branch.
