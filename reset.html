Git Reset Command

What is Git Reset?

Git reset is a command in Git that is used to reset the current branch to a specific state. It is commonly used to undo changes, unstage files, or move the HEAD pointer to a different commit.

Git Reset Commands

To undo changes in the working directory and staging area for a specific file, you can use:

git reset HEAD <file>

This command unstages the changes for the specified file, but it does not discard the changes in the working directory.

To undo changes in the working directory and staging area for all files, you can use:

git reset HEAD

This command unstages all changes, but it does not discard the changes in the working directory.

To move the HEAD pointer to a specific commit and reset the staging area and working directory to match that commit, you can use:

git reset --hard <commit>

This command resets the HEAD pointer to the specified commit and discards all changes in the staging area and working directory.

Example Usage

Let's say you have made some changes and staged them for commit, but you decide to undo the staging for a specific file:

  1. Run the reset command:
    2. git reset HEAD <file>
  2. This will unstage the changes for the specified file, but keep the changes in the working directory.

If you want to completely discard the changes and move the HEAD pointer to a specific commit:

  1. Run the reset command with the --hard option:
    2. git reset --hard <commit>
  2. This will reset the staging area and working directory to match the specified commit, discarding all changes.
