Git Stash Command

What is Git Stash?

Git stash is a command in Git that allows you to temporarily save your local modifications without committing them to a branch. It is particularly useful when you need to switch branches or perform other operations without committing your current changes.

Git Stash Command

To stash your changes, you can use the following command:

git stash

This command will stash (save) your modifications in a "stash" and revert your working directory to the state of the last commit.

Later, when you want to apply the stashed changes back to your working directory, you can use:

git stash apply

This command will apply the most recently stashed changes to your working directory.

If you have multiple stashes, you can specify which one to apply using:

git stash apply stash@{n}

where n is the index of the stash you want to apply.

Example Usage

Let's consider a scenario where you are working on a feature branch, but you need to switch to another branch to fix a critical bug. Instead of committing your changes or losing them, you can stash them:

git stash

After fixing the bug and committing the changes, you can switch back to your feature branch and apply the stashed changes:

git stash apply

This way, you can seamlessly switch between branches without losing your work.
