Git Merge Conflict and Commands

What is a Merge Conflict?

A merge conflict in Git occurs when two branches have diverged, and Git cannot automatically merge the changes. This typically happens when two branches have modified the same part of a file, and Git is unable to determine which change to accept.

Resolving Merge Conflicts

When a merge conflict occurs, you need to resolve it manually. Git marks the conflicted areas in the affected files, and you must edit the files to resolve the conflicts. Here are the steps to resolve a merge conflict:

  1. Identify the conflicted files using git status.
  2. Open the conflicted files in a text editor.
  3. Locate and resolve the conflict markers (<<<<<<<, =======, and >>>>>>>).
  4. Save the changes.
  5. Add the resolved files to the staging area using git add <resolved-file>.
  6. Complete the merge by committing the changes using git commit.

Example

Let's say you are merging changes from a feature branch into the main branch, but a merge conflict occurs in a file named example.txt. To resolve the conflict:

  1. Open example.txt in a text editor.
  2. Locate and resolve the conflict markers.
  3. Save the changes.
  4. Add the resolved file to the staging area:
    5. git add example.txt
  5. Commit the changes:
    6. git commit -m "Resolved merge conflict"

After resolving the conflict and committing the changes, the merge is complete.
